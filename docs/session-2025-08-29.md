## New Plugin Flow (Entry Points)
- Group: `pdf2zh_next.translators`
- Each entry point should point to a callable that registers translators via `TranslatorRegistry.register(...)`.
- Fallback: if the entry point targets a module, the loader tries to auto-match `*Translator` and `*Settings` classes.

## Runtime Precheck
- Validates plugin `Requires-Python` and all `Requires-Dist` using `importlib.metadata` + `packaging`.
- Specially checks plugin’s required `pdf2zh-next` range.
- Incompatible plugins are skipped with actionable `pip install` suggestions in logs.

## How To Use
- Install example Sungrow plugin locally (editable):
  - `pip install -e ./examples/pdf2zh-plugin-sungrow`
- Audit installed plugins:
  - `pdf2zh-plugin-doctor`
- Programmatic discovery:
  - Python: `from pdf2zh_next.translator import load_plugins, TranslatorRegistry; load_plugins(); print(TranslatorRegistry.list_custom_translators())`
- Run smoke tests:
  - `pytest -q`

## Migration Notes
- Directory-based plugin loading is deprecated and removed:
  - No scanning of `~/.pdf2zh/plugins`, `./pdf2zh_plugins`, `PDF2ZH_PLUGIN_DIR`, or in-repo `translator/plugins`.
- Create a standalone Python package with entry point instead.
